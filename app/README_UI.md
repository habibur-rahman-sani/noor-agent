# Phase 1 — ওয়েব UI + ব্যাকগ্রাউন্ড সার্ভিস

## কী যোগ হলো

- `server.py` — FastAPI সার্ভার, তোমার existing `supervisor.py` (teams/agents অপরিবর্তিত)
  কেই ব্যবহার করে। REST (`/api/chat`) আর WebSocket (`/ws/chat`, স্ট্রিমিং) দুটোই আছে।
- `static/index.html` — চ্যাট UI (কোনো বিল্ড স্টেপ লাগে না, প্লেইন HTML/CSS/JS)।
  বাম পাশে সব টিমের তালিকা, যেই টিম কাজ করছে তার পাশে আলো জ্বলবে।
- `static/manifest.json` — এর ফলে ফোনে/ল্যাপটপে ব্রাউজার থেকে "Install App" করলে
  হোমস্ক্রিন/ডেস্কটপ আইকন হিসেবে বসে যাবে (PWA)।
- `requirements.txt`-এ `fastapi`, `uvicorn`, `websockets` যোগ হয়েছে।

## চালানো (লোকাল টেস্ট)

```bash
pip install -r requirements.txt --break-system-packages
uvicorn server:app --host 0.0.0.0 --port 8000
```

ব্রাউজারে খোলো: `http://localhost:8000` (একই নেটওয়ার্কের অন্য ডিভাইস থেকে হলে
সার্ভার-মেশিনের লোকাল আইপি দিয়ে, যেমন `http://192.168.x.x:8000`)।

**লগইন:** সার্ভার `0.0.0.0`-এ চলে বলে (একই নেটওয়ার্কের যে কেউ অ্যাক্সেস করতে
পারে), পুরো UI/API HTTP Basic Auth দিয়ে সুরক্ষিত (`auth.py`)। ব্রাউজার প্রথমবার
একটা native ইউজারনেম/পাসওয়ার্ড popup দেখাবে। ইউজারনেম/পাসওয়ার্ড `.env`-এর
`AGNO_UI_USERNAME`/`AGNO_UI_PASSWORD`-এ — খালি রাখলে সার্ভার প্রথমবার চালু
হওয়ার সময় নিজে থেকে একটা র‍্যান্ডম পাসওয়ার্ড জেনারেট করে `.env`-এ লিখে দেয় ও
লগে (`journalctl -u agno-system`) দেখায়। (ISO থেকে বুট করা সিস্টেমে এটা
বিল্ড-টাইমেই জেনারেট হয়ে ডেস্কটপের `README-FIRST.txt`-এ লেখা থাকে।)

## "UI বন্ধ থাকলেও চলবে" — ব্যাকগ্রাউন্ডে/সবসময় চালু রাখা

### দ্রুত উপায় (টেস্টের জন্য)
```bash
nohup uvicorn server:app --host 0.0.0.0 --port 8000 > server.log 2>&1 &
```

### প্রোডাকশন উপায় (Linux, systemd) — recommended
`/etc/systemd/system/agno.service` বানাও:
```ini
[Unit]
Description=Agno Supervisor Web Server
After=network.target

[Service]
WorkingDirectory=/path/to/agno_system
ExecStart=/path/to/agno_system/venv/bin/uvicorn server:app --host 0.0.0.0 --port 8000
Restart=always
EnvironmentFile=/path/to/agno_system/.env

[Install]
WantedBy=multi-user.target
```
তারপর:
```bash
sudo systemctl enable --now agno
```
এখন সার্ভার বুটেই চালু হবে, ক্র্যাশ করলে অটো-রিস্টার্ট হবে, আর তুমি ব্রাউজার/UI
বন্ধ করলেও এটা চলতেই থাকবে — যেকোনো সময় ব্রাউজারে গিয়ে আবার কানেক্ট করলেই
চলবে। (ফোন থেকে অ্যাক্সেস করতে চাইলে ঘরের বাইরে থেকেও পৌঁছাতে হলে
Tailscale/Cloudflare Tunnel-এর মতো কিছু দিয়ে সার্ভারটা এক্সপোজ করতে হবে —
চাইলে সেটাও পরের ধাপে সেট করে দিতে পারি।)

## এখন যা কাজ করে

- টেক্সট চ্যাট (স্ট্রিমিং সহ)
- কোন টিম উত্তর দিচ্ছে তার লাইভ ইন্ডিকেটর
- session অটো-তৈরি হয় ও ব্রাউজারে (localStorage) মনে থাকে — পেজ রিফ্রেশ করলেও
  সেশন হারাবে না (MongoDB memory-র সাথে মিলিয়ে)
- মোবাইল/ডেস্কটপ ব্রাউজার — দুটোতেই রেসপন্সিভ, PWA হিসেবে ইনস্টলযোগ্য

## Phase 2 — ভয়েস (এইমাত্র যোগ হলো)

বাম সাইডবারে "মোড" সেকশনে এখন দুটো অপশন কাজ করে:

- **শুধু টেক্সট** — আগের মতোই, কোনো ভয়েস নেই।
- **টেক্সট + ভয়েস** — 🎤 বাটনে চাপুন → কথা বলুন → আবার চাপুন থামাতে → বলা কথা
  টেক্সট হয়ে ইনপুট বক্সে বসবে (এডিট করা যাবে বা কীবোর্ড দিয়ে নতুন করে লেখাও যাবে),
  তারপর Enter/পাঠান চাপলেই যাবে। সুপারভাইজারের উত্তর টেক্সট আকারে আসার পর
  নিচে "🔊 শুনুন" চিপ দেখাবে ও অটো-প্লে হবে (আবার শুনতে চিপে ক্লিক করুন)।
- **অ্যাভাটার** — এখনো ডিসেবল করা আছে, Phase 3-এ চালু হবে।

### নতুন সিস্টেম-নির্ভরতা

STT অডিও ডিকোড করতে `ffmpeg` লাগবে:
```bash
sudo apt install ffmpeg
```

### প্রথমবার ব্যবহারে

STT (`faster-whisper`, "small" মডেল) আর TTS (`facebook/mms-tts-ben` / `mms-tts-eng`)
মডেলগুলো **প্রথমবার ব্যবহারের সময় ইন্টারনেট থেকে ডাউনলোড হবে** (কয়েকশো MB, একবারই),
এরপর লোকালি ক্যাশ থেকে চলবে — তখন আর ইন্টারনেট না থাকলেও ভয়েস মোড কাজ করবে, শুধু
টেক্সট চ্যাটের জন্য এখনো OpenRouter (ইন্টারনেট) লাগবে।

CPU-তে চলবে, তবে প্রথম কয়েকটা রিকোয়েস্টে একটু ধীর লাগতে পারে (মডেল লোড হওয়ার সময়)।
GPU থাকলে `voice.py`-তে `device="cuda"` করে দিলে অনেক দ্রুত হবে।

## Phase 3 — অ্যাভাটার (এইমাত্র যোগ হলো)

মোড সুইচারে এখন **"অ্যাভাটার"** অপশন চালু। এটা নেয় Phase 2-এর STT/TTS-কেই,
শুধু সাথে একটা নরম-আলো দেওয়া গোলাকার মুখ (চোখ পলক ফেলে, breathe করে) — যখন
সুপারভাইজার বলে, TTS অডিওর ভলিউম রিয়েল-টাইমে বিশ্লেষণ করে (Web Audio API,
ব্রাউজারেই হয়, সার্ভারে কোনো চাপ পড়ে না) মুখ/আলো সেই অনুযায়ী নড়ে — কথা বলার
মতো একটা জীবন্ত অনুভূতি দেয়।

### সৎভাবে বলে রাখা — এটা কীসের মতো, কীসের মতো না

এটা **amplitude-driven lip animation** — অডিওর জোর/ছন্দ অনুযায়ী মুখ নড়ে,
আসল phoneme-নির্ভর নিখুঁত লিপ-সিঙ্ক না (যেমন "ব" আর "আ"-তে আলাদা মুখের শেপ না)।
এটার সুবিধা: **রিয়েল-টাইমে চলে, GPU লাগে না, কোনো দেরি হয় না।**

একটা ভারী বিকল্প আছে — SadTalker/Wav2Lip-এর মতো ওপেন-সোর্স মডেল দিয়ে
ছবি/ভিডিও থেকে ফটোরিয়ালিস্টিক মুখ বানিয়ে phoneme-accurate লিপ-সিঙ্ক করা — কিন্তু
সেটা প্রতিটা উত্তরের জন্য একটা ভিডিও **জেনারেট** করতে হয় (GPU ছাড়া অনেক ধীর,
প্রতি উত্তরে কয়েক সেকেন্ড থেকে মিনিট লাগতে পারে) — চ্যাটের মতো তাৎক্ষণিক
কথোপকথনের জন্য এটা উপযুক্ত না। আপনার যদি ভালো GPU থাকে আর দেরি মেনে নেওয়া
যায়, এটা পরে আলাদা "উচ্চমানের ভিডিও রিপ্লাই" ফিচার হিসেবে যোগ করে দেওয়া যাবে।

## এখনো যা বাকি (পরের ধাপ)

- **Phase 4** — বাইরে থেকে অ্যাক্সেস (Tailscale/Cloudflare Tunnel), auth/লগইন,
  Docker প্যাকেজিং, ব্যাকগ্রাউন্ড সার্ভিস হিসেবে পাকাপাকি ডিপ্লয়
- (ঐচ্ছিক, ভবিষ্যতে) GPU থাকলে SadTalker/Wav2Lip দিয়ে ফটোরিয়ালিস্টিক
  ভিডিও-অ্যাভাটার আপগ্রেড
- **Phase 4** — সত্যিকারের বাইরে থেকে অ্যাক্সেস (Tailscale/Cloudflare Tunnel),
  auth/লগইন যোগ (এখন কোনো authentication নেই — লোকাল নেটওয়ার্কেই রাখাই নিরাপদ),
  Docker-এ প্যাকেজ করা

## একটা গুরুত্বপূর্ণ নোট

`server.py`-র স্ট্রিমিং অংশ (`supervisor.arun(..., stream=True)`) Agno-র সাধারণ
API প্যাটার্ন অনুযায়ী লেখা হয়েছে, কিন্তু এই স্যান্ডবক্সে ইন্টারনেট/agno প্যাকেজ
ইনস্টল করে সরাসরি রান করে টেস্ট করা সম্ভব হয়নি (নেটওয়ার্ক বন্ধ ছিল)। তোমার
মেশিনে `.env` বসিয়ে প্রথমবার চালানোর সময় যদি স্ট্রিমিং অংশে এরর আসে
(যেমন Agno-র নতুন/পুরনো ভার্সনে event শেপ একটু আলাদা), আমাকে সেই এরর মেসেজটা
দিলে আমি এক টার্নে ঠিক করে দেব — কোডে ইচ্ছাকৃতভাবে try/except fallback রাখা
আছে যাতে স্ট্রিমিং ব্যর্থ হলেও সাধারণ (নন-স্ট্রিমিং) উত্তর অন্তত পাওয়া যায়।
